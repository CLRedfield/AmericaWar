## 1. 游戏定位

这是一个以 **KX 风格“最大规模美国内战”** 为题材的多人联机网页战争小游戏。

玩家从 8 个固定势力中选择 1 个，在抽象化的美国格点地图上争夺工业区、铁路枢纽、港口、油田与首都，通过：

- 回合资源结算
- 政治点行动抉择
- 征兵与调兵
- 节点建设
- 国策成长
- 多人联机博弈

最终达成统一美国或取得霸权。

这个游戏不是要做成 HOI4 那种超复杂大战略，而是要做成：

- 规则易懂
- 节奏清晰
- 联机好玩
- 阵营差异鲜明
- 1 局约 45~90 分钟

的网页战争游戏。

## 2. 核心设计原则

### 2.1 固定 8 势力，不再扩编

首版国家永久锁定为以下 8 个：

1. 美利坚合众国（麦克阿瑟军政府）
2. 美利坚联合工团
3. 美利坚联盟国
4. 美利坚宪政国
5. 新英格兰
6. 美利坚太平洋国
7. 西部指挥军区
8. 德克萨斯临时政府

这样可以让开发范围稳定，不会因为不停加势力而无限膨胀。

### 2.2 地图看着大，底层规则保持简单

底层核心仍然是：

- 格点节点控制
- 工业产出
- 步兵驻军与移动
- 政治点行动
- 简洁战斗结算
- 国策差异化

### 2.3 强调“行动节奏”而不是“单位微操”

因为每回合只获得 **2 点政治点**，且行动成本会递增，所以“这一回合做什么”本身就是最核心的策略。

## 4. 地图系统

### 4.1 地图形式

地图采用 **“格点 + 邻接关系”** 结构。

- 每个点代表一个战略区域，而不是严格州界
- 相邻节点之间可移动部队
- 特殊线路（铁路、海运、山口）以后可用额外连线补充

### 4.2 节点基础属性

```ts
type MapNode = {
  id: string;                // 节点ID
  name: string;              // 节点名称
  owner: FactionId | null;   // 当前控制者
  industry: number;          // 工业值
  troops: number;            // 驻军数
  isCapital: boolean;        // 是否首都
  terrain: TerrainType;      // 地形
  tags: string[];            // 港口 / 铁路 / 油田 / 工业城市等
  adjacent: string[];        // 相邻节点
};
```

### 4.3 首版地形类型

| 地形 | 效果 |
|---|---|
| 平原 | 无修正 |
| 城市 | 守方 +20% |
| 工业区 | 工业值高，争夺重点 |
| 山地 | 守方 +30% |
| 森林 | 守方 +15% |
| 湿地 | 攻方 -10% |
| 港口 | 便于做贸易 / 海运扩展 |
| 铁路枢纽 | 后续可扩展战略机动 |
| 油田 | 后续可给德州等阵营额外收益 |

## 6. 资源系统（稳定度已移除）

首版只保留 **2 种核心资源**：

| 资源 | 说明 | 主要用途 |
|---|---|---|
| 金钱 `$` | 由节点工业提供 | 征兵、维护、建设 |
| 政治点 `PP` | 每回合基础获得 | 行动、点国策 |

### 6.1 首版推荐参数

- 政治点基础收入：**每回合 2 点**
- 政治点可储存：**可以**
- 政治点上限：**10**
- 金钱收入：控制节点工业值总和
- 步兵维护费：每个步兵 `0.25`
- 征兵价格：`1 元 = 1 步兵`
- 工业上限：普通节点 `5`，重要工业节点可到 `6`
- 建设工业基础消耗：`5 PP`

> 说明：虽然每回合只有 2 点政治点，但只要允许跨回合储存，玩家依然可以攒点做“建设工业”或“多连动回合”。

## 7. 回合流程

每名玩家在自己回合开始时，按以下顺序结算：

### 7.1 回合开始结算

1. **获得金钱**  
   金钱 = 所有己方节点工业值总和

2. **支付维护**  
   每个步兵消耗 `0.25` 元

3. **财政不足处理**  
   若钱不够，则自动解散部分部队，直到维护费可支付。建议优先解散：
   - 非首都节点
   - 非前线节点
   - 工业低、战略价值低的后方节点

4. **获得政治点**  
   每回合基础获得 **2 PP**，且不能超过上限 `10`

5. **进入行动阶段**

## 8. 行动系统

### 8.1 核心行动

首版保留 4 种核心行动：

| 行动 | 基础消耗 | 说明 |
|---|---:|---|
| 放兵 | 1 PP | 只能在首都执行，`1 元 = 1 步兵` |
| 移动步兵 | 1 PP | 本回合所有未移动过的己方步兵，各可移动 1 次 |
| 点国策 | 1 PP | 执行一个尚未点过的国策 |
| 建设工业 | 5 PP | 选择一个己方节点，工业值 +1 |

### 8.2 行动递增消耗

延续你原规则的关键机制：

> 若本回合已经执行了 `k` 次行动，则下一次行动需要额外消耗 `k` 点政治点。

公式：

```text
行动总消耗 = 行动基础消耗 + 本回合已执行行动数
```

示例：

| 第几次行动 | 额外消耗 | 如果执行“放兵”总消耗 |
|---:|---:|---:|
| 第 1 次 | 0 | 1 PP |
| 第 2 次 | 1 | 2 PP |
| 第 3 次 | 2 | 3 PP |
| 第 4 次 | 3 | 4 PP |

这会自然形成：

- 少量高价值行动
- 攒点爆发回合
- 不会无限操作

的节奏。

## 9. 移动与占领

### 9.1 移动规则

- 只能移动到相邻节点
- 每个步兵每回合只能移动 1 次
- 同一行动中，所有未移动过的己方部队都可一起移动一次
- 后续可加入铁路 / 山口特殊移动，但首版先不做

### 9.2 占领规则

- 移入 **空节点**：直接占领
- 移入 **敌方节点**：触发战斗
- 若战斗胜利：获得该节点控制权

### 9.3 首都规则

- 首都是唯一可征兵点
- 首都失守不会立刻灭国
- 若某国：
  - 已失去首都，且
  - 地图上已无任何其他节点，
  - 则判定灭亡

## 10. 战斗系统

首版采用简单直接的结算方式。

### 10.1 战斗力计算

- 攻方战斗力 = `进攻兵力 × (1 + 进攻修正)`
- 守方战斗力 = `驻守兵力 × (1 + 防守修正)`

### 10.2 地形修正

| 地形 | 守方加成 |
|---|---:|
| 平原 | 0% |
| 城市 | +20% |
| 山地 | +30% |
| 森林 | +15% |
| 湿地 | +10% |

### 10.3 战斗结算

- 若攻方战斗力 > 守方战斗力，则攻方占领节点
- 若守方战斗力 ≥ 攻方战斗力，则守方守住节点

建议幸存兵力算法：

```text
幸存兵力 = max(1, ceil(胜方战斗力 - 负方战斗力))
```

优点：

- 规则直观
- 易于前端展示
- 易于和地形 / 国策联动

## 11. 国策系统

### 11.1 设计原则

首版每国做 **5 个国策** 即可。

统一树形结构：

- A：起始可点
- B：起始可点
- C：需要 A
- D：需要 B
- E：需要 C 和 D

### 11.2 可复用效果池

国策只做容易实现的数值类效果：

- 立即获得金钱
- 立即获得政治点
- 首都获得步兵
- 指定节点工业 +1
- 永久降低征兵价格
- 永久降低维护费
- 永久提高进攻修正
- 永久提高防守修正
- 永久每回合额外获得金钱
- 永久每回合额外获得政治点
- 永久降低建设工业基础消耗

### 11.3 8 国国策方向建议

| 势力 | 国策方向 |
|---|---|
| USA | 军政集中、首都防御、快速调动 |
| CSA | 工业动员、工人民兵、生产扩张 |
| AUS | 南方动员、低成本征兵、快速扩张 |
| CON | 法统整合、防守反击、稳健运营 |
| NEN | 港口贸易、精兵、防御工事 |
| PAC | 西海岸资本、财政增长、后期强化 |
| WDC | 山地要塞、铁路调度、消耗战 |
| TEX | 石油收益、区域机动、突击作战 |

## 13. 胜利条件

首版建议只做两种胜利方式：

### 13.1 歼灭胜利

地图上最后一个存活国家获胜。

### 13.2 霸权胜利

某国控制全图 **60% 以上节点** 时，直接获胜。

> 这个门槛可以避免多人局拖得过长。
