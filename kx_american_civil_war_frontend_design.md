# KX 最大规模美国内战联机网页小游戏：前端设计文档

版本：v0.2 修订版  
目标：为“8 势力固定版”的回合制多人联机格点战争游戏提供前端结构、页面布局、组件拆分、交互流程与数据接口建议。

> 本版根据你的最新要求修订：
> - **基础政治点收入统一为每回合 2 点**
> - **势力固定为 8 个**
> - **去除稳定度系统相关 UI 与数据**
> - **前端明确支持格点图 / 节点占位图展示**

---

## 1. 前端设计目标

这个项目的前端需要解决 3 个核心问题：

1. **地图要一眼能看懂**  
   玩家必须能快速识别：谁控制哪里、哪些点能打、哪些点是工业点、哪些点是首都。

2. **回合操作不能迷路**  
   征兵、移动、建设工业、点国策，都要被压进清晰的操作流里。

3. **联机状态必须可信**  
   玩家要随时知道：当前是谁的回合、还剩多少时间、我的操作有没有提交成功。

## 2. 推荐技术栈

### 2.1 首版推荐

| 模块 | 技术 |
|---|---|
| 框架 | React + TypeScript |
| 构建 | Vite |
| 状态管理 | Zustand 或 Redux Toolkit |
| 地图渲染 | SVG + HTML Overlay |
| 联机通信 | Socket.IO 或 Colyseus |
| UI | Tailwind CSS + shadcn/ui |
| 动画 | Framer Motion |
| 数据格式 | JSON 配置表 |

### 2.2 为什么首版用 SVG 地图

格点战争游戏非常适合 SVG：

- 节点、连线、区域块都容易画
- 悬停 / 点击 / 高亮很方便
- 与 React 状态绑定简单
- 100 个以内节点性能完全足够
- 以后想做更炫效果再换 PixiJS 也不迟

## 6. TopStatusBar

### 显示内容

从左到右建议显示：

1. 游戏名 / 房间名
2. 当前回合数
3. 当前行动玩家
4. 倒计时
5. 自己的金钱
6. 自己的政治点
7. 下一次行动额外 PP
8. 节点数 / 工业总值
9. 网络连接状态
10. 结束回合按钮

### 示例

```txt
裂旗战争 | 第 7 回合 | 当前：联合工团 | 01:24
$ 18 | PP 6/10 | 下一行动 +2 PP | 节点 11 | 工业 19
[结束回合]
```

### 状态颜色建议

| 状态 | 表现 |
|---|---|
| 轮到自己 | 顶栏发亮，结束回合按钮高亮 |
| 不是自己 | 顶栏变暗，显示“等待某某行动” |
| 计时小于 15 秒 | 倒计时闪烁 |
| 断线重连中 | 右侧显示黄色网络警告 |

> 注意：TopStatusBar 已去除“稳定度”显示。

## 8. MapCanvas

## 8.1 地图层级

SVG 地图建议分 7 层：

```txt
<svg>
  Layer 1: BackgroundMapShape       美国底图
  Layer 2: RegionBlocks             大区色块 / 格点底盘
  Layer 3: Connections              节点连线
  Layer 4: NodesBase                节点圆点 / 城市图标
  Layer 5: ControlRings             控制方颜色环
  Layer 6: TroopLabels              兵力与工业数字
  Layer 7: EffectsAndHighlights     选中 / 可移动 / 可攻击高亮
</svg>
```

### 8.2 节点视觉编码

| 元素 | 表现 |
|---|---|
| 控制者 | 节点外圈颜色 |
| 工业 | 节点下方小数字或齿轮图标 |
| 兵力 | 节点中央数字 |
| 首都 | 星形标记 |
| 油田 | 油滴图标 |
| 港口 | 锚点图标 |
| 可移动 | 蓝色脉冲外圈 |
| 可攻击 | 红色脉冲外圈 |
| 选中 | 白色粗描边 |
| 战斗中 | 震动或火花动画 |

### 8.3 格点占位图展示模式

为了更接近你要的“格点图展示每个点占位”，前端建议提供一个 **战术网格视图**：

- 平时显示美化后的战略地图
- 点击“网格视图”按钮后，切换成简化格点图
- 格点图中每个格子直接显示：节点缩写 / 所属势力 / 工业值 / 首都标识

示意：

```txt
[SEA PAC 2] [IDA WDC 1] [MON WDC 1] [NDK WDC 1] [MIN CSA 2] [WIS CSA 2]
[SFO PAC 3C][NEV WDC 1] [UTA WDC 1] [WYO WDC 1] [OMA CON 1] [CHI CSA 4C]
[LAX PAC 3] [AZN WDC 1] [COL WDC 2C][KNS CON 1] [STL CON 2] [IND CON 2]
```

这个模式很适合：

- 开发测试
- 新手教学
- 直播讲解
- 调整平衡时快速核对初始布局

### 8.4 地图交互

#### 鼠标悬停节点

显示 Tooltip：

```txt
芝加哥
控制：美利坚联合工团
地形：城市 / 工业区
工业：4
驻军：12
相邻：威斯康星、底特律、奥马哈、印第安纳、圣路易斯
```

#### 点击己方节点

- 选中节点
- 右侧显示节点详情
- 高亮可移动目标
- 显示可执行操作：征兵 / 移动 / 建设工业

#### 点击敌方相邻节点

如果已选中己方部队节点：

- 打开 BattlePreviewModal
- 展示预计结果
- 玩家确认后进攻

#### 地图操作

- 左键拖拽空白区域：平移
- 滚轮：缩放
- 双击节点：居中

## 16. 本版前端文档的最终结论

前端应围绕以下 4 个关键词来做：

- **格点清晰**：节点归属、工业、兵力、首都必须一眼可读
- **操作直接**：玩家能很快完成征兵、移动、建设、点国策
- **联机可信**：所有回合与操作状态都要实时可见
- **低认知负担**：因为已经去掉稳定度系统，所以 UI 可以更专注于地图、资源、战斗与国策

如果按这个版本推进，前端首版会比较像：

- 一个简洁的大地图联机桌游界面
- 一个可切换“美术地图 / 网格图”的战争面板
- 一个信息量适中、但很容易上手的 KX 内战网页游戏前端
